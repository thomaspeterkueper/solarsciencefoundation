# SSF Document Source Architecture - Consensus Pending

Status: pending external consensus
Created: 2026-07-08
Owner: Solar Science Foundation

## Purpose

SSF will need a document-source model for learning modules, learning paths, exercises, references, and archive-based educational content.

This model must not be defined by SSF alone. The canonical ownership boundaries must first be clarified with:

- kueper-knowledge-graph
- kueper.com
- overtime-archive.org

## Current Working Assumption

SSF is primarily a learning platform.

SSF should:

- consume canonical document references
- transform source material into learning modules
- store learning progress
- provide exercises and learning paths
- expose progress and unlock data to partner projects

SSF should not become the canonical owner of general KUEPER documents unless a later cross-repository decision explicitly assigns a document type to SSF.

- overtime-archive.org: external-tasks/open/OTA-REQ-20260708-document-architecture-consensus.md

## Decision Gate

Do not finalize any SSF document-source schema until the external repositories have clarified:

1. canonical document ownership
2. KG document reference model
3. kueper.com document ID and versioning model
4. OTA source, mirror, and in-universe document boundaries
5. how SSF may cite, derive from, and expose source documents

## Temporary Rule

Until consensus is reached, SSF may use document references in learning content only as provisional references. It must not create a competing canonical document registry.
