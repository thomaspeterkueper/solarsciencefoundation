## Purpose

Die KG-interne Wissenspyramide (L0–L4) soll als didaktisches Strukturprinzip in die
SSF-Lernoberfläche übertragen werden — ohne die technischen Labels zu verwenden.

Lernende sollen erleben, dass Wissen aufeinander aufbaut: Beobachtung → Begriff →
Modell → Anwendung → Geschichte. Die Architektur bleibt unsichtbar; die Erfahrung
von Tiefe und Zusammenhang wird sichtbar.

---

## Hintergrund

Der KUEPER Knowledge Graph definiert fünf Ebenen:

```
L0 Foundation
L1 Knowledge
L2 Models
L3 Applications
L4 Narratives
```

SSF konsumiert bereits KXF-Daten mit Dependency-Graphen und Modul-IDs.
Was fehlt: eine didaktische Übersetzung dieser Struktur in die Lernoberfläche.

---

## Requested Content

### 1. Mapping: KG-Ebenen → SSF-Lernerfahrung

| KG-Ebene | SSF-Entsprechung (intern) | Lernerfahrung (sichtbar) |
|----------|--------------------------|--------------------------|
| L0 | Beobachtungsanker | "Was habe ich wahrgenommen?" |
| L1 | Begriff / Konzept | "Wie nennt man das?" |
| L2 | Modell / Erklärung | "Warum verhält es sich so?" |
| L3 | Anwendung / Transfer | "Was kann ich damit machen?" |
| L4 | Narrativ / Kontext | "Wo taucht das in der Welt auf?" |

### 2. Sichtbarkeitsprinzip für die LearnMap

- Jedes Modul zeigt: welche Module es voraussetzt, welche es freischaltet
- Labels wie "L0", "Foundation", "Physik" bleiben verborgen
- Tiefenindikator (bestehende Fortschrittslogik) reflektiert implizit die Ebenenstruktur

### 3. Designregel

Narrative Inhalte (L4) in SSF-Modulen illustrieren Konzepte.
Sie definieren keine Konzepte.
Diese Regel ist im Redaktionsleitfaden zu verankern.
