# SSF-0004 — SSF Magazine: Candidate Scanner implementieren

## Kontext

ECO-ARC-0009 (accepted) definiert SSF Magazine als Sub-Modul
mit Pull-not-Push-Architektur. Erster Schritt: Candidate Scanner.

## Architektur (aus ECO-ARC-0009)

```
Candidate Scanner (wöchentlich)
  → zieht Themen-Kandidaten aus:
    - SSF Lernpfade (neue Cluster)
    - OTA-Dokumente (neue Archiv-Einträge)
    - NOXIA (neue Missionen / Freischaltungen)
    - Externe Quellen (RSS, ingenieur.de, SWR Wissen, FR)
  → erstellt Kandidaten-Liste mit Priorität
  → übergibt an Draft Generator (Claude API)
```

## Minimaler erster Schritt

Statische Kandidaten-Liste aus SSF-Lernpfaden generieren:
- Alle Pfade mit `status: 'prototype'` und mind. 3 Kapiteln
- Sortiert nach Cluster-Neuheit
- Export als JSON für manuelle Kuration

## Dateipfad

`/ssf-magazine/scanner/candidates.ts`

## Aufwand

Niedrig für statische Version, mittel für automatisierten Scanner.
